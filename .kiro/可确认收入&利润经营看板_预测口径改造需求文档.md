# 《可确认收入&利润经营看板》预测口径改造需求文档

## 1. 需求背景
当前《可确认收入&利润经营看板》页面结构、交互和模块布局已基本稳定，但当前页面中的“可确认收入”和“销售额”仍更偏向实际统计口径，未完全体现增值类服务业务中的预测属性。

结合业务特点，安防增值类服务的收入确认通常具有跨期确认特征，订单成交金额不会在成交当期一次性全部确认为收入，而是需要依据服务生效周期、套餐时长和确认规则，在后续多个周期内逐步确认。

因此，本看板需要从“经营结果统计页”调整为“经营预测分析页”，重点呈现：
- 销售额预测
- 可确认收入预测
- 成本预测
- 利润预测
- 利润率预测
- 套餐盈利能力预测

本次调整重点不是重做页面布局，而是统一指标口径、业务定义、图表说明、模块表达，使页面真正符合“预测型经营驾驶舱”的定位。

当前页面标题和布局虽已接近经营驾驶舱，但指标表达上仍容易被理解为：
- 实际销售额
- 实际可确认收入
- 实际成本
- 实际利润

这与当前业务目标不一致。

本看板定位调整为：

**基于增值类服务跨期确认逻辑的收入与利润预测经营看板**

用于从日 / 周 / 月不同粒度观察未来经营表现，辅助运营团队进行：
- 收入预测判断
- 利润预测判断
- 套餐结构优化
- 经营风险识别
- 运营策略参考

## 3. 核心业务逻辑说明

### 3.1 增值类服务业务特征
当前业务为安防增值类服务，主要包括：
- 云存
- 云+4G
- 4G
- AI
- 云+AI

该类业务存在以下关键特征：
1. 用户购买套餐后形成订单销售额
2. 销售额不会直接等于当期收入
3. 收入需依据服务生效与确认规则逐步确认
4. 月包、季包、年包等不同套餐具有不同确认周期
5. 因此需要将销售额与可确认收入分开展示，并以预测方式进行经营分析

### 3.2 销售额预测定义
销售额预测是指在当前统计周期内，预计形成的订单成交金额。

销售额预测反映业务成交规模，但不直接等于当期收入。

页面中需要区分：
- 觅睿销售额预测
- 客户销售额预测（如有）
- 总销售额预测

当前页面如只保留“觅睿销售额”，则需明确其为：

**觅睿销售额预测**

### 3.3 可确认收入预测定义
可确认收入预测是指基于订单销售额预测结果，结合服务生效与收入确认规则，在当前统计周期内预计可以确认的收入金额。

可确认收入预测具有以下特征：
1. 不等于订单销售额预测
2. 受套餐时长影响
3. 受服务开始时间影响
4. 会跨日 / 跨周 / 跨月逐步确认
5. 是利润预测计算的核心收入口径

### 3.4 可确认收入预测构成
总可确认收入预测由两部分组成：
- 觅睿收款可确认收入预测
- 客户收款可确认收入预测

公式：

**总可确认收入预测 = 觅睿收款可确认收入预测 + 客户收款可确认收入预测**

### 3.5 跨期确认逻辑说明
由于增值类服务通常按服务周期持续生效，因此一笔订单的金额可能在多个周期内逐步确认为收入。

示例：
- 月包：通常在较短周期内确认
- 季包：收入可能跨多个自然月确认
- 年包：收入需要跨更长周期逐步确认

因此：
- 销售额预测体现的是订单成交规模
- 可确认收入预测体现的是当前统计周期预计确认的收入规模

页面必须明确体现这两者差异。

### 3.6 成本预测定义
成本预测包括以下部分：
- 服务器成本预测
- 流量成本预测
- 支付平台手续费预测
- 觅睿分成成本预测
- 客户分成成本预测

成本预测与销售额预测、可确认收入预测共同构成利润预测逻辑。

### 3.7 利润预测定义
利润预测采用以下口径：

**利润预测 = 可确认收入预测 - 成本预测**

### 利润率预测
**利润率预测 = 利润预测 / 可确认收入预测**

利润口径必须与可确认收入预测保持一致，不能直接用销售额预测计算利润率。

**收入·成本·利润 主趋势分析**

- 可确认收入预测
- 觅睿销售额预测

- 利润预测

**总可确认收入预测 - 各项成本预测 = 利润预测**

- 哪类套餐产品利润预测更高

#### 要求
模块标题、副标题、tooltip、摘要卡统一调整为预测口径，例如：

- 分成成本预测趋势
- 流量成本预测趋势或说明
- 成本占比说明

若存在“日粒度下流量成本无日维度”的特殊逻辑，保留原规则不变，但文案需统一为预测口径。

### 4.8 异常监控调整
异常监控区表达的是：
- 当前预测经营结果中的风险提示

因此需要从“实际异常”表述调整为“预测风险/预测预警”表达。

例如：
- 利润率预测低于预设阈值
- 单设备流量成本预测环比异常上升
- 手续费率预测环比异常上升

## 5. 时间粒度口径保留规则

当前已确认的时间交互逻辑保留，不做改变，仅将统计口径统一为预测版本。

### 5.1 KPI 规则
- 日粒度：截止日期当天预测值
- 周粒度：截止日期所在周截至截止日期的预测累计值
- 月粒度：截止日期所在月截至截止日期的预测累计值

### 5.2 趋势图规则
- 日粒度：不自动拓展，严格按所选时间范围按天展示预测趋势
- 周粒度：自动拓展近 8 周，按周聚合预测趋势
- 月粒度：自动拓展近 6 个月，按月聚合预测趋势

### 5.3 页面说明
页面需明确区分：
- KPI：当前统计周期预测累计值
- 趋势图：历史/未来观察窗口中的预测趋势
